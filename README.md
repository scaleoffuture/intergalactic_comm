# intergalactic_comm
The second game of mine in Russian that was created as the independent project for Python's course in HSE. The collaboration with AI.

## The game was created based on the following criteria (in RU):

1) Использовать любые библиотеки / импортированные функции запрещено.	

2) Для проверки кода предоставлен файл в формате .ipynb	

3) Создана блок-схема, которая описывает сюжет. (файл формата .pdf)	

4) В итоге работы представлен цельный сюжет с минимум пятью различными концовками (на достижение которых может влиять пользователь своим выбором).	

5) В коде использовано хотя бы 4 условных конструкций. Например, сюжет такой: грабитель убегает от преследования и выбирает, свернуть направо или налево — если он свернет направо, то его задержит полиция, а если он свернет направо, то встретит мафию, которая заберет у него все награбленное. Таких разветвлений в сюжете может быть и больше.	

6) В коде использован хотя бы один цикл while. Например, сюжет такой: грабитель пытается подобрать пароль к сейфу, пока не закончится время. Если время вышло, то его могут поймать жильцы, а если грабитель успел вовремя, то он получает драгоценности из сейфа.	

7) В коде использован хотя бы один цикл for. Например, сюжет такой: на этаже есть 4 двери и грабитель проверяет каждую на предмет наличия за дверью того, что можно украсть. Он может даже зайти в каждую комнату и осмотреться.	

8) В коде использовано хотя бы два метода списков / два метода множеств / один метод списков и один метод множеств. Например, списком в данном случае может быть сумка грабителя, в которую он складывает украденное.	

9) В коде использован хотя бы один готовый словарь. Например, в виде словаря может быть представлена адресная книга, из которой грабитель выбирает себе потенциальную жертву.	

10) В коде создается хотя бы один новый словарь. Например, если грабителя поймала полиция, то она может записать в словарь названия статей, которые он нарушил, и наказание по каждой статье. 

11) Для упрощения работы кода создана минимум одна собственная функция (чтобы не дублировать повторяющиеся части кода).

12) Итоговый сюжет (со всеми выборами пользователя) и получившаяся концовка автоматически записаны в файл (файл .txt).

13) Код написан, потому что он нужен для работы, а не просто, чтобы "выполнить критерий".
